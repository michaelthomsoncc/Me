Richard Develyn

Inventor, CTO & Co-Founder of CloudTrade

Introduction

Richard Develyn is a technology innovator and co-founder of CloudTrade Technologies Limited, a UK-based company founded in 2010 specializing in electronic document processing, e-invoicing, and accounts payable automation. As the former Chief Technology Officer (CTO) and founding director, he played a pivotal role in developing the company's proprietary software, including Gramatica, which leverages AI and rules-based technology for high-accuracy data extraction from documents like PDFs. Develyn is the inventor of the core patent underpinning CloudTrade's technology: US Patent No. 9,558,295 B2, titled "System for Data Extraction and Processing." This page provides a detailed overview of his background and the invention, based on available public information.

Abstract and Purpose

The patent describes a system designed to automatically extract and process data from electronic documents, such as PDFs, which often lack structured context (e.g., no embedded metadata for easy parsing). Traditional methods like optical character recognition (OCR) can struggle with varying layouts, fonts, or visual elements, leading to errors in data interpretation. This invention addresses these issues by combining a data acquisition engine for raw extraction with a sophisticated processing engine that applies rules and AI-like algorithms to interpret the data meaningfully. The goal is to convert unstructured document data into a structured format (e.g., XML or CSV) for seamless integration into business systems, such as invoice processing software. This enables automation, reduces manual data entry, and minimizes errors, which is particularly useful for high-volume tasks like handling supplier invoices in finance departments.

How the Invention Works

The system operates in two main stages: data acquisition and data processing. Here's a step-by-step breakdown:

  1. Data Acquisition Engine:
    • This component scans the electronic document and extracts individual "glyphs" (characters or symbols) along with their spatial and visual properties. These properties include:
    • Location on the page (e.g., x-y coordinates).
    • Font type, size, and style.
    • Background color, shading, or other visual cues (e.g., bold text for headers).
    • Unlike basic text extraction, this engine captures contextual visual information, which helps in understanding the document's structure. For example, it can differentiate a table header from body text based on shading or positioning.
    • The engine sources both character-level data (text content) and pixel-level information (visual rendering) from the document file.
  2. Processing Engine:
    • This is the core of the invention, using a set of predefined rules and a "backward tracking search" algorithm to interpret the extracted data.
    • Rules: These are hierarchical guidelines that define how to analyze the data. At least one rule specifies an "anchor point"—a reliable, standardized element in the document (e.g., a postcode in a specific format like "AB12 3CD" for UK addresses). Anchor points serve as starting references to "anchor" the interpretation process.
    • Rules describe relationships between glyphs and their properties. For instance, a rule might say: "If text is bold and located above a table, treat it as a header."
    • Multiple anchor points can be used if needed, and rules are applied in a hierarchy to find the "best fit" interpretation.
    • Backward Tracking Search Algorithm: This AI-inspired method handles ambiguity by working like a logical reasoning engine:
    • It defines "axioms" (known facts from the data) and "hypotheses" (possible interpretations).
    • The algorithm applies parts of a hypothesis to the axioms and tests if they match. If a match fails, it "backtracks"—reverting to earlier steps, discarding invalid assumptions, and trying alternatives.
    • This process continues until a valid meaning is ascribed to the data, ensuring robustness even for complex or inconsistent documents.
    • Sectioning for Efficiency: Long documents can be divided into logically uncorrelated sections (e.g., separate invoice pages), with anchor points identified in each. This prevents exponential processing delays.
    • Output Formatting: Once interpreted, the data is formatted into a user-defined structure:
    • XML: With specific tags (e.g., 12345).
    • CSV: Sorted lists (e.g., "InvoiceNumber,SupplierName,TotalAmount").
    • The system can timestamp outputs for versioning and handle multiple document types.

Technical Advantages and Innovations

High Accuracy: By incorporating visual properties (beyond just text), the system achieves near-100% accuracy for supported document types, outperforming template-based or pure OCR methods that fail on layout variations.

Adaptability: Rules allow the system to handle diverse documents without custom programming per format. The backward tracking ensures it adapts to edge cases, like misaligned text.

Scalability and Efficiency: Document sectioning and hierarchical rules reduce computational load. It's cloud-compatible, aligning with CloudTrade's SaaS model.

Error Reduction: Automation minimizes human intervention, cutting costs in processes like e-invoicing (CloudTrade claims processing billions in invoice value).

Limitations Addressed: Prior art (e.g., German Patent DE101622156 for basic structure analysis) is improved upon by adding AI rules and visual context for semantic interpretation.

This patent forms the foundation of CloudTrade's products, such as Gramatica and Cloud Capture, which apply these methods to real-world scenarios like PDF invoice processing for clients in finance and supply chains. The technology has enabled partnerships (e.g., with Cegedim and Valtatech) and contributed to CloudTrade's growth until its acquisition.

Legacy and Influence

Develyn's invention has had a lasting impact on intelligent document processing (IDP), influencing subsequent technologies like those at CloudConnect, where evolved versions of this rules-based AI approach continue to drive accuracy in document automation.
